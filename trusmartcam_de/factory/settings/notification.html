Notifications

Benachrichtigungen informieren Benutzer über CAD- und Werkzeugzuordnungsfehler, die im System auftreten. Sie stellen sicher, dass Probleme zeitnah kommuniziert werden, damit geeignete Maßnahmen ergriffen werden können.

E-Mail-Benachrichtigung

Die E-Mail-Benachrichtigungsfunktion ermöglicht es Benutzern, automatisierte E-Mail-Benachrichtigungen für CAD- und Werkzeugzuordnungsfehler zu konfigurieren und zu empfangen. Nach der Konfiguration sendet das System automatisch E-Mails an die angegebenen Empfänger, wann immer ein relevantes Ereignis auftritt, um sicherzustellen, dass Probleme zeitnah bearbeitet werden.

Um E-Mail-Benachrichtigungen zu konfigurieren, navigieren Sie zu: Werk > Einstellungen > Benachrichtigungen

Die Seite Benachrichtigungen besteht aus den folgenden Abschnitten:

Notification settings

Der Abschnitt Benachrichtigungseinstellungen steuert, wie E-Mail-Benachrichtigungen generiert und an Empfänger zugestellt werden.

  • E-Mail-Benachrichtigungen aktivieren - Aktiviert das System, um Benachrichtigungen basierend auf den konfigurierten Bedingungen zu senden.

  • Empfänger von Benachrichtigungen - Gibt die E-Mail- Adresse(n) der Empfänger an. Mehrere Empfänger können durch Trennung mit einem Komma (,) oder einem Leerzeichen hinzugefügt werden.

  • Nachrichten gesammelt verschicken - Fasst mehrere Benachrichtigungen zu einer einzigen E-Mail zusammen. Nachrichten, die innerhalb eines definierten Zeitintervalls generiert werden, werden zusammengefasst, was dazu beiträgt, das E-Mail-Volumen zu reduzieren und Unübersichtlichkeit zu vermeiden.

  • Vorschau der Werkzeugsimulation beifügen, wenn zutreffend + - Fügt der Benachrichtigungs-E-Mail eine Werkzeugzuordnungs-Simulationsvorschau hinzu, wenn verfügbar. Dies ist nützlich, um werkzeugbezogene Probleme direkt aus der E-Mail zu überprüfen.

Das Einschließen der Simulationsvorschau kann die E-Mail-Größe erheblich vergrößern.

E-Mail-Einstellungen

Der Abschnitt E-Mail-Einstellungen definiert die Konfiguration des ausgehenden Mail- Servers, den das System verwendet, um E-Mail-Benachrichtigungen zuzustellen. Diese Einstellungen müssen korrekt konfiguriert sein, damit Benachrichtigungen erfolgreich gesendet werden können.

  • SMTP-Server - Gibt die SMTP-Serveradresse an, die zum Senden ausgehender E-Mails verwendet wird (z.B. smtp.gmail.com). Diese Information ist normalerweise im Konto- oder Einstellungsbereich des Mail-Clients des Benutzers verfügbar.

  • SMTP-Schnittstelle - Definiert die Portnummer, die vom Mail- Server zum Senden von E-Mails über das Internet verwendet wird. Häufige Werte sind 465 (SSL) und 587 (TLS).

  • Transportprotokoll - Gibt das Protokoll an, das zum sicheren Senden von E-Mails von der Anwendung an den Mail-Server verwendet wird (z.B. SSL oder TLS).

  • Server benötigt Authentifizierung - Aktiviert die Authentifizierung für den SMTP-Server. Wenn aktiviert, werden der Benutzername und das Passwort vom Server validiert, bevor E-Mails gesendet werden.

  • Benutzername - Gibt das E-Mail-Konto an, das zur Authentifizierung mit dem SMTP-Server und zum Senden von Benachrichtigungen verwendet wird.

  • Passwort - Gibt das Passwort an, das mit dem E-Mail- Konto für die Authentifizierung verknüpft ist.

  • Sender address - Gibt die Absender-E-Mail-Adresse an, die den Empfängern angezeigt wird, wenn Benachrichtigungen empfangen werden.

  • Testen…​ - Sendet eine Test-E-Mail zur Überprüfung der Mail-Server- Konfiguration.

Klicken Sie auf Testen…​, um die Einrichtung zu validieren. Eine Bestätigungsmeldung wird angezeigt, wenn der Test erfolgreich ist; andernfalls wird eine Fehlermeldung angezeigt, die die Ursache des Fehlers angibt.

  • E-Mail-Benachrichtigungen werden im interaktiven Modus nicht ausgelöst (z.B. beim Bearbeiten oder manuellen Speichern eines Teils).

  • Die E-Mail-Zustellung wird im Hintergrund verarbeitet und kann verzögert sein.

  • Benachrichtigungen werden möglicherweise nicht zugestellt, wenn das System oder die Engine gestoppt wird, bevor die Verarbeitung abgeschlossen ist.
