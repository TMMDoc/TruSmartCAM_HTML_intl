Jobs importieren

Import aus CSV oder XLSX

Sie können Jobs auch in gängigen Textformaten importieren

  • Klicken Sie auf File → Open → Jobs_importieren, der Dialog "Ordner öffnen" erscheint. Wählen Sie C:\Program Files\Metamation\Praxis\Samples\Jobs\C0\C0.csv.

  • Wenn alle Teile im CSV-Speicherort oder in den Lookup-Ordnerverzeichnissen vorhanden sind, erscheint das Fenster neuer Job mit einer Liste aller Teile, die den CSV-Feldern entsprechen.

  • Fehlende Teile werden ignoriert und erscheinen nicht im Fenster neuer Job.

  • Anfänglich ist der Teilestatus Nascent.

  • Klicken Sie auf OK, nun werden die Teile in die Teilebibliothek importiert. CAD-Validierung, Werkzeugbestückung für Biege- und Schneidwerkzeuginstanz erfolgen, die im Bildschirm Pmonitor überwacht werden können.

Leeres Teil für die fehlenden CAD-Dateien erstellen

Beim Importieren von Jobs mithilfe einer Tabellenkalkulation (.csv oder .xlsx) können Sie manchmal Zeilen haben, die auf CAD-Dateien verweisen, die im System nicht verfügbar sind.

  • Um dies zu handhaben, gehen Sie zu Werk → Einstellungen → Job, aktivieren Sie unter Tabellenblattimport die Option Leeres Teil für die fehlenden CAD-Dateien erstellen

Wenn diese Option ausgewählt ist:

  • Wenn eine CAD-Datei für eine Zeile während des Job-Imports fehlt, erstellt das System automatisch ein leeres Teil anstelle der fehlenden CAD-Datei.

  • Im Fenster neuer Job werden diese fehlenden Teile gelb hervorgehoben, um anzuzeigen, dass die zugehörige CAD-Datei fehlt.

  • Das fehlende Teil hat weiterhin seine zugeordneten Felder (wie Teilename, Material, Dicke, Menge usw.) aus dem CSV. Sein Teilestatus wird jedoch als Nascent angezeigt, was bedeutet, dass es nur ein leerer Platzhalter ohne tatsächliche Geometrie ist.

Dies stellt sicher, dass der Job-Import auch ohne Fehler aufgrund fehlender CAD-Dateien fortgesetzt werden kann — Sie können das leere Teil später aktualisieren oder durch die korrekte CAD-Datei ersetzen, wenn diese verfügbar wird.

Dialog „Neuer Job“ nicht anzeigen, wenn das Tabellenblatt fehlerfrei ist

Werk → Einstellungen → Job → Tabellenblattimport→ Dialog „Neuer Job“ nicht anzeigen, wenn das Tabellenblatt fehlerfrei ist

  • Wenn dieses Kontrollkästchen aktiviert ist, erstellt das System automatisch den neuen Job, ohne den Dialog "Neuer Job" anzuzeigen, wenn die Tabellenkalkulation keine Fehler aufweist (keine fehlenden Teile, fehlenden Daten oder ungültigen Daten).

  • Wenn dieses Kontrollkästchen deaktiviert ist, wird der Dialog "Neuer Job" immer zur Überprüfung und Bestätigung durch den Benutzer angezeigt, auch wenn die Tabellenkalkulation keine Fehler aufweist.

Prefer raw-material column over material + thickness

Werk → Einstellungen → Job→Prefer Raw-Material column over Material + Thickness

  • Wenn dieses Kontrollkästchen aktiviert ist, erwartet das System, dass die Tabellenkalkulation die Spalte "Raw-Material" verwendet (z. B. Steel-20). Dies bedeutet, dass Materialtyp und Dicke in einem einzigen Wert in einer Spalte kombiniert werden sollten.

  • Wenn dieses Kontrollkästchen deaktiviert ist, erwartet das System, dass die Tabellenkalkulation separate Spalten für Material und Dicke hat (z. B. Steel und 2.00). Das System kombiniert sie während des Imports intern.

Überwachungsordner mit CSV/XLSX

  • In Werk → Einstellungen→ Beobachten aktivieren Sie „Live-Ordner“ aktivieren und Auch Unterordner beobachten.

  • Legen Sie den Pfad des Überwachungsordners fest (UNC oder zugeordnetes Laufwerk).

  • In Tabellenblattimport aktivieren Sie Tabellenblatt aus „Live-Ordner“ importieren.

  • Wählen Sie Jobs_importieren aus den Optionen für den Tabellenkalkulationsimport.

  • Kopieren Sie C:\Program Files\Metamation\Praxis\Samples\Jobs\C0\C0.csv und fügen Sie es in den Überwachungsordner ein.

  • Das System überprüft das CSV und versucht, den Job automatisch zu erstellen.

  • Wenn die CAD-Datei fehlt, wird kein Job erstellt, da die erforderlichen CAD-Daten nicht gefunden werden.

CAD-Lookup-Verzeichnis & rekursive CAD-Suche - nicht konfiguriert

  • In Werk → Einstellungen → Job → Tabellenblattimport, wenn kein CAD lookup directories festgelegt ist und Rekursive CAD-Suche verwenden nicht verwendet wird, sucht das System nicht automatisch nach CAD-Dateien.

  • Wenn das Kontrollkästchen Leeres Teil für die fehlenden CAD-Dateien erstellen aktiviert ist, wird der Job dennoch erstellt, auch wenn CAD-Dateien fehlen.

  • In diesem Fall werden alle Teile im Job als Nascent-(leere) Teile erstellt — das bedeutet, sie haben keine Geometrie, da die CAD-Dateien nicht gefunden werden konnten.

CAD-Lookup-Verzeichnis & rekursive CAD-Suche - konfiguriert

Wenn CAD-Suchverzeichnisse auf (C:\Program Files\Metamation\Praxis\Samples) gesetzt ist und Rekursive CAD-Suche verwenden aktiviert ist unter Werk → Einstellungen → Job → Tabellenblattimport:

  • Das System sucht automatisch im angegebenen Ordner und allen Unterordnern nach den erforderlichen CAD-Dateien, die in der Tabellenkalkulation erwähnt werden.

  • Leeres Teil für die fehlenden CAD-Dateien erstellen ist ebenfalls aktiviert. Wenn also eine CAD-Datei immer noch nicht gefunden wird, erstellt das System ein leeres Teil als Platzhalter.
