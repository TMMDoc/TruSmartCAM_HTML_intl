ツーリングパネル

このセクションでは、ツーリングインスタンスの管理に使用される、 アウトプットのエクスポート、 Check Tooling、ツーリングをスキップする、 ベース平面として設定する…、ツーリングのコピー…、 およびその他のオプションなど、ツーリング関連のアクションについて説明します。

読み取り専用モードで開いて見ます
選択したツーリングを読み取り専用モードで開きます。

Check-out the document for editing
選択したツーリングの編集を有効にします。

アウトプットのエクスポート
設定された送信先に選択したツーリング出力をエクスポートします。

Check Tooling
変更を加えずに既存のツーリングを検証し、検証が失敗した場合は ステータスを更新します。

ツーリングのコピー…
選択した参照ツーリングからpunch、シーケンス、およびback gaugeセットアップを コピーし、実行可能な場合は他のソリューションに適用します。

ベース平面として設定する…
選択したbendまたはfoldツーリングを、切断ソリューション生成の 参照フラットサイズとしてマークします。

Re-tool…​
他のツーリングに影響を与えずに、パートの選択したツーリングを再処理します。

ツーリングをスキップする
選択したツーリングを抑制して、さらなる処理と出力生成から除外します。

前へ / 次へ / パーツ
前または次のツーリング、または親パートに移動します。

並べ替えとフィルタ
クイックリファレンスのために、選択したツーリングと似た特性を共有する 他のツーリングを表示します。

アウトプットのエクスポート

このオプションは、*ファクトリー設定*で設定された送信先に パート出力をエクスポートします。(Bend Settingsおよび Cut Settingsを参照)

選択したツーリング操作に関連付けられたツーリングのみがエクスポートされ、 パート全体はエクスポートされません。

エクスポートされるファイルは操作タイプによって異なります。BendおよびFold操作の場合、 NCファイルとレポートが生成されます。CutおよびPunch操作の場合は、パートファイルのみが エクスポートされます。

Check Tooling

  • CAM Settingsまたはベンドツールのいずれかに変更があった場合、パートを Re-toolする必要はありません。

  • 代わりに、Check Toolingを使用して、変更を加えずに既存のツーリングを検証します。

  • 検証が失敗した場合、そのインスタンスのツーリングステータスが更新され、 関連する出力ファイルが削除されます。

  • Check Toolingは、*エラーを無視する*を使用して手動で エラーが上書きされたパートも検出し、必要に応じてそれらのステータスを エラーに更新します。

ツーリングのコピー…

Copy toolingを使用すると、1つのソリューションから他のソリューションに ベンドツーリングをすばやく適用でき、時間と労力を節約できます。

これにより、選択した参照ツーリング(ソリューション)からpunch、シーケンス、および back gaugeセットアップがコピーされ、同じ設定をすべての他のソリューションに 適用しようとします。

  • コピーされたセットアップがターゲットマシンに実行可能な場合は、保存されて使用されます。

  • セットアップが実行可能でない場合は、そのマシンの元のツーリングがそのまま残ります。

たとえば、異なるマシンに対して異なるツーリングがある場合があります:

  • ここでは、Machine 5230SX 6AがGooseneck punchを使用し、Machine 5085X 6A B23が Pointed punchを使用しています。

Machine 5230SX 6Aで*ツーリングのコピー…*を使用すると、 Gooseneckセットアップをすべての他のマシンにコピーしようとします。

可能な場合は、同じpunch、シーケンス、およびback gaugeの設定が適用されます。 コピーされたセットアップを使用できない場合は、元のツーリングがそのまま残ります。

Copy toolingを適用した後、Gooseneckセットアップは実行可能なすべての他のマシンに 正常に適用されています。

ベース平面として設定する…

*ベース平面として設定する…*機能は、切断ソリューションの生成に 参照フラットサイズとして使用されるbendまたはfoldツーリングを定義します。 ツーリングがBase-Flatとしてマークされると、TruSmartCAMはそのマシンに基づいて パートのフラットサイズを更新し、それに応じて切断ソリューションを再生成します。 Base-FlatツーリングはUIで青い枠線で表示されます。

異なるベンドマシンでは、同じパートに対してわずかにフラットサイズが異なる場合があります。 ツーリングをBase-Flatとして設定すると、TruSmartCAMはそのマシンのフラットサイズを マスター参照として使用し、切断ソリューションを自動的に再計算します。

自動base flat選択

ファクトリ設定の *曲げ加工中に平面修正を適用する*が有効になっている場合:

  • TruSmartCAMは、パートインポート時に(最大ベンド長テーブルを含む)bendツーリングを 自動的にBase-Flatとして設定します。

  • 選択したbendツーリングにエラーが発生した場合、または実行できない場合、 TruSmartCAMはインテリジェントに切り替わり、別の有効なbendツーリングをBase-Flatとして 設定します。これにより、手動介入なしでフラットサイズ参照と切断ソリューションが 常に一貫性があり有効であることが保証されます。

拡張されたbase flat管理

TruSmartCAMは、bendまたはfoldツーリングの編集を保存する際に、 *Praxisに保存する*ダイアログでBase-Flatオプションを提供します。 これらのオプションは、OKステータスのツーリングに対してのみ表示され、 フラットサイズと切断ソリューションが正確に保たれるようにします。

ベース平面として設定して、切断ソリューションを計算します

このオプションは、ツーリングをBase-Flatとして初めてマークし、 対応する切断ソリューションを自動的に生成するために使用されます。

表示される条件:

  • ツーリングが現在Base-Flatとしてマークされていない。

  • ユーザーが編集のためにツーリングをチェックアウトし、変更を保存している。

このオプションを使用するには、bendまたはfoldツーリングをチェックアウトし、 ベンドパラメータやフラットサイズの変更など、必要な変更を行います。 Ctrl+Sを押して変更を保存します。Praxisに保存する ダイアログで、 *ベース平面として設定して、切断ソリューションを計算します*を選択し、 *OK*をクリックします。TruSmartCAMは、ツーリングをBase-Flatとして設定し、 更新されたフラットサイズを使用して切断ソリューションを生成します。

ベース平面を更新して、切断ソリューションを再計算します

このオプションは、ツーリングがすでにBase-Flatとしてマークされており、 編集されていて、フラットサイズと切断ソリューションを再生成する必要がある場合に使用されます。

表示される条件:

  • ツーリングがすでにBase-Flatとしてマークされている。

  • ユーザーがチェックアウトし、編集し、変更を保存している。

既存のBase-Flatツーリングを編集する場合、必要に応じてベンドパラメータまたは フラットサイズのジオメトリを更新します。*Praxisに保存する*ダイアログで、 *ベース平面を更新して、切断ソリューションを再計算します*を選択します。 TruSmartCAMは、Base-Flatツーリングを更新し、関連するすべての切断ソリューションを 再生成して最新の変更を反映します。

一度に1つのbendツーリングのみをBase-Flatとして設定できます。

Base-Flatが変更されると、既存の切断ソリューションは自動的に再処理されます。

この機能は、TruSmartCAM内のbendおよびfoldツーリングの両方で使用できます。

再ツーリング…

このオプションは、パートの選択したツーリングを再処理します。

Toolingパネルで必要なツーリングを右クリックし、 *再ツーリング…*をクリックします。

続行する前に確認ダイアログが表示されます。

*はい*をクリックして再ツーリングを続行するか、*いいえ*をクリックして 操作をキャンセルします。

選択したツーリングのみが再ツーリングされます。パートに関連付けられた他のツーリングは 影響を受けません。選択したツーリングに関連する既存の出力は削除され、 再ツーリングプロセス中に再生成されます。

ツーリングデータが古い、不正確な場合、またはマシンやツーリング構成に変更が加えられた場合に このオプションを使用します。

エラーを無視する

  • ツールされたインスタンスに警告が表示され、それでも続行したい場合は、 *エラーを無視する*をクリックして警告を上書きできます。

  • これにより、ツーリングの問題があった場合でも、システムはそのインスタンスの 出力ファイルをエクスポートできるようになります。

*Ignore Errors*は、警告を安全に無視できることが確実な場合にのみ使用してください。 実際のエラーを上書きすると、マシンの損傷や安全上のリスクにつながる可能性があります。

ツーリングの抑制/抑制解除

  • マシンが非アクティブであるか、特定のパートを特定のマシンで処理したくない場合は、 *ツーリングをスキップする*を使用して、そのマシンでのネスティングから そのパートを除外します。

  • 抑制後、ツールされたインスタンスにマウスを合わせると、次のようなメッセージが 表示されます:「Tooling suppressed by <User name> on <Date Time>」。

  • Suppress toolingは、そのマシンとパートの出力ファイルを削除します。

  • *ツーリングをスキップしない*を使用して、そのマシンでのパートの ツーリングステータスを復元します。

  • 抑制解除されると、それぞれの出力ファイルが再度生成されます。
