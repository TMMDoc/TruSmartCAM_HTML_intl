Import jobs

csvまたはxlsxからインポート

一般的なテキスト形式でジョブをインポートすることもできます。

  • *File → Open → ジョブのインポート*をクリックすると、Open Folderダイアログが表示されます。`C:\Program Files\Metamation\Praxis\Samples\Jobs\C0\C0.csv`を選択します。

  • csvの場所または検索フォルダーディレクトリ内にすべてのパーツが存在する場合、csvフィールドに一致するすべてのパーツをリストした*新しいジョブ*ウィンドウが表示されます。

  • 不足しているパーツは無視され、*新しいジョブ*ウィンドウには表示されません。

  • 最初、パーツのステータスはNascentです。

  • *OK*をクリックすると、パーツがパートライブラリにインポートされます。CAD検証、曲げとカットツーリングインスタンスのツーリングが実行され、*Pmonitor*画面で監視できます。

欠けているCADファイルのブランク材パーツを作成する

スプレッドシート（.csvまたは.xlsx）を使用してジョブをインポートする際、システム内で利用できないCADファイルを参照する行がある場合があります。

  • これに対処するには、ファクトリー → 設定 → *ジョブ*に移動し、*スプレッドシートのインポート*の下で*欠けているCADファイルのブランク材パーツを作成する*を有効にします。

このオプションが選択されている場合：

  • ジョブインポート中にいずれかの行のCADファイルが不足している場合、システムは不足しているCADファイルの代わりに空白パーツを自動的に作成します。

  • *新しいジョブ*ウィンドウでは、これらの不足しているパーツは、関連付けられたCADファイルが不足していることを示すために黄色でハイライト表示されます。

  • 不足しているパーツには、csvからマップされたフィールド（パーツ名、材料、厚さ、数量など）がまだ含まれています。しかし、そのパーツステートはNascentとして表示されます。これは、実際の形状がまだない空のプレースホルダーであることを意味します。

これにより、CADファイルが不足しているためにエラーが発生することなく、ジョブインポートを続行できます — 利用可能になったら、後で空白パーツを正しいCADファイルで更新または置き換えることができます。

スプレッドシートにエラーがない場合は新しいジョブダイアログを表示しない

ファクトリー → 設定 → ジョブ → スプレッドシートのインポート→ スプレッドシートにエラーがない場合は新しいジョブダイアログを表示しない

  • このチェックボックスが選択されている場合、スプレッドシートにエラーがなければ（不足しているパーツ、不足している日付、または無効なデータがない）、システムはNew Jobダイアログを表示せずに新しいジョブを自動的に作成します。

  • このチェックボックスがクリアされている場合、スプレッドシートにエラーがなくても、ユーザーのレビューと確認のためにNew Jobダイアログが常に表示されます。

材料+厚さより原材料列を優先

ファクトリー → 設定 → ジョブ→材料+厚さより原材料列を優先

  • このチェックボックスが選択されている場合、システムはスプレッドシートが原材料列を使用することを期待します（例：Steel-20）。これは、材料タイプと厚さを1つの列の単一の値に結合する必要があることを意味します。

  • このチェックボックスがクリアされている場合、システムはスプレッドシートに材料と厚さの別々の列があることを期待します（例：Steelと2.00）。システムはインポート中に内部でそれらを結合します。

CSV/XLSXを使用した監視フォルダー

  • ファクトリー → 設定→ 見る*で「ライブフォルダ」を有効にする*と*サブフォルダも見る*をチェックします。

  • 監視フォルダーパス（UNCまたはマップされたドライブ）を設定します。

  • *スプレッドシートのインポート*で、*スプレッドシートを「ライブフォルダ」からインポートする*をチェックします。

  • Spreadsheet Importオプションから*ジョブのインポート*を選択します。

  • `C:\Program Files\Metamation\Praxis\Samples\Jobs\C0\C0.csv`をコピーして監視フォルダーに貼り付けます。

  • システムはCSVをチェックし、ジョブを自動的に作成しようとします。

  • CADファイルが不足している場合、必要なCADデータが見つからないため、ジョブは作成されません。

CAD検索ディレクトリと再帰CAD検索 - 未設定

  • ファクトリー → 設定 → ジョブ → *スプレッドシートのインポート*で、*CAD検索ディレクトリ*が設定されておらず、*繰り返しCAD検索を使用する*が使用されていない場合、システムはCADファイルを自動的に検索しません。

  • *欠けているCADファイルのブランク材パーツを作成する*チェックボックスが選択されている場合、CADファイルが不足していてもジョブは作成されます。

  • この場合、ジョブ内のすべてのパーツはnascent（空白）パーツとして作成されます — つまり、CADファイルが見つからなかったため、形状がありません。

CAD検索ディレクトリと再帰CAD検索 - 設定済み

CAD検索ディレクトリ*が(C:\Program Files\Metamation\Praxis\Samples)に設定され、*繰り返しCAD検索を使用する*が*ファクトリー → 設定 → ジョブ → *スプレッドシートのインポート*で有効になっている場合：

  • システムは、指定されたフォルダーとそのすべてのサブフォルダーを自動的に検索して、スプレッドシートに記載されている必要なCADファイルを探します。

  • *欠けているCADファイルのブランク材パーツを作成する*も有効になっているため、CADファイルがまだ見つからない場合、システムはプレースホルダーとして空白パーツを作成します。
