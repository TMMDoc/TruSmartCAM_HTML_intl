Notifications

Notifications는 시스템 내에서 발생하는 CAD 및 툴링 오류에 대해 사용자에게 알립니다. 적절한 조치를 취할 수 있도록 문제가 신속하게 전달되도록 보장합니다.

Email notification

Email notification 기능을 사용하면 사용자가 CAD 및 툴링 오류에 대한 자동 이메일 알림을 구성하고 받을 수 있습니다. 구성되면 시스템은 관련 이벤트가 발생할 때마다 지정된 수신자에게 자동으로 이메일을 보내 문제가 신속하게 해결되도록 보장합니다.

이메일 알림을 구성하려면 다음으로 이동합니다: 공장 > 설정 > 알림

알림 페이지는 다음 섹션으로 구성됩니다:

Notification settings

알림 설정 섹션은 이메일 알림이 생성되고 수신자에게 전달되는 방법을 제어합니다.

  • 이메일 알림 활성화 - 시스템이 구성된 조건에 따라 알림을 보낼 수 있도록 활성화합니다.

  • 알림 수신자 - 수신자의 이메일 주소를 지정합니다. 쉼표(,) 또는 공백으로 구분하여 여러 수신자를 추가할 수 있습니다.

  • 메시지 일괄 전송 - 여러 알림을 단일 이메일로 결합합니다. 정의된 시간 간격 내에 생성된 메시지가 함께 그룹화되어 이메일 볼륨을 줄이고 혼잡을 방지하는 데 도움이 됩니다.

  • 가능한 경우 툴링 시뮬레이션 미리보기 첨부 + - 사용 가능한 경우 알림 이메일에 툴링 시뮬레이션 미리보기를 첨부합니다. 이는 이메일에서 직접 툴링 관련 문제를 검토하는 데 유용합니다.

시뮬레이션 미리보기를 포함하면 이메일 크기가 크게 증가할 수 있습니다.

메일 설정

메일 설정 섹션은 시스템이 이메일 알림을 전달하는 데 사용하는 발신 메일 서버 구성을 정의합니다. 알림이 성공적으로 전송되려면 이러한 설정을 올바르게 구성해야 합니다.

  • SMTP 서버 - 발신 이메일을 보내는 데 사용되는 SMTP 서버 주소를 지정합니다(예: smtp.gmail.com). 이 정보는 일반적으로 사용자의 메일 클라이언트의 계정 또는 설정 섹션에서 사용할 수 있습니다.

  • SMTP 포트 - 메일 서버가 인터넷을 통해 이메일을 보내는 데 사용하는 포트 번호를 정의합니다. 일반적인 값은 465(SSL) 및 587(TLS)입니다.

  • 전송 프로토콜 - 애플리케이션에서 메일 서버로 이메일을 안전하게 보내는 데 사용되는 프로토콜을 지정합니다(예: SSL 또는 TLS).

  • 서버 인증 필요 - SMTP 서버에 대한 인증을 활성화합니다. 활성화하면 이메일이 전송되기 전에 서버에서 사용자 이름과 비밀번호의 유효성을 검사합니다.

  • 사용자 이름 - SMTP 서버로 인증하고 알림을 보내는 데 사용되는 이메일 계정을 지정합니다.

  • 비밀번호 - 인증에 사용되는 이메일 계정과 연결된 비밀번호를 지정합니다.

  • Sender address - 알림이 수신될 때 수신자에게 표시되는 발신자 이메일 주소를 지정합니다.

  • 테스트…​ - 메일 서버 구성을 확인하기 위해 테스트 이메일을 보냅니다.

*테스트…​*를 클릭하여 설정을 확인합니다. 테스트가 성공하면 확인 메시지가 표시되고, 그렇지 않으면 실패 원인을 나타내는 오류 메시지가 표시됩니다.

  • 이메일 알림은 대화형 모드에서는 트리거되지 않습니다(예: 편집 또는 수동으로 부품 저장).

  • 이메일 전달은 백그라운드에서 처리되며 지연될 수 있습니다.

  • 처리가 완료되기 전에 시스템 또는 엔진이 중지되면 알림이 전달되지 않을 수 있습니다.
