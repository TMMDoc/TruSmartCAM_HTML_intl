Notificações

As notificações informam os usuários sobre erros de CAD e ferramental que ocorrem dentro do sistema. Elas garantem que os problemas sejam comunicados prontamente para que as ações apropriadas possam ser tomadas.

Notificação por e-mail

O recurso de notificação por e-mail permite que os usuários configurem e recebam alertas automatizados por e-mail para erros de CAD e ferramental. Uma vez configurado, o sistema envia automaticamente e-mails para os destinatários especificados sempre que um evento relevante ocorrer, garantindo que os problemas sejam tratados prontamente.

Para configurar notificações por e-mail, navegue até: factory > settings > Notifications

A página Notifications consiste nas seguintes seções:

Configurações de notificação

A seção Notification settings controla como os alertas de e-mail são gerados e entregues aos destinatários.

  • Enable email notifications - Habilita o sistema para enviar notificações com base nas condições configuradas.

  • Notification recipients - Especifica o(s) endereço(s) de e-mail dos destinatários. Múltiplos destinatários podem ser adicionados separando-os com uma vírgula (,) ou um espaço.

  • Send messages in batch - Combina múltiplas notificações em um único e-mail. Mensagens geradas dentro de um intervalo de tempo definido são agrupadas, o que ajuda a reduzir o volume de e-mails e evita sobrecarga.

  • Attach tooling simulation preview where applicable - Anexa uma pré-visualização da simulação de ferramental ao e-mail de notificação quando disponível. Isso é útil para revisar problemas relacionados a ferramental diretamente a partir do e-mail.

Incluir a pré-visualização da simulação pode aumentar significativamente o tamanho do e-mail.

Mail settings

A seção Mail settings define a configuração do servidor de e-mail de saída que o sistema usa para entregar notificações por e-mail. Essas configurações devem estar corretas para que as notificações sejam enviadas com sucesso.

  • SMTP server - Especifica o endereço do servidor SMTP usado para enviar e-mails de saída (por exemplo, smtp.gmail.com). Esta informação geralmente está disponível na seção de conta ou configurações do cliente de e-mail do usuário.

  • SMTP port - Define o número da porta usada pelo servidor de e-mail para enviar e-mails pela internet. Valores comuns incluem 465 (SSL) e 587 (TLS).

  • Transport protocol - Especifica o protocolo usado para enviar e-mails de forma segura da aplicação para o servidor de e-mail (por exemplo, SSL ou TLS).

  • Server needs authentication - Habilita a autenticação para o servidor SMTP. Quando habilitada, o nome de usuário e a senha são validados pelo servidor antes que os e-mails sejam enviados.

  • User-name - Especifica a conta de e-mail usada para autenticar com o servidor SMTP e enviar notificações.

  • Password - Especifica a senha associada à conta de e-mail usada para autenticação.

  • Sender address - Especifica o endereço de e-mail do remetente que é exibido para os destinatários quando as notificações são recebidas.

  • Test…​ - Envia um e-mail de teste para verificar a configuração do servidor de e-mail.

Clique em Test…​ para validar a configuração. Uma mensagem de confirmação é exibida se o teste for bem-sucedido; caso contrário, uma mensagem de erro é exibida indicando a causa da falha.

  • Notificações por e-mail não são acionadas no modo interativo (por exemplo, durante a edição ou ao salvar manualmente uma peça).

  • A entrega de e-mail é processada em segundo plano e pode sofrer atrasos.

  • As notificações podem não ser entregues se o sistema ou engine for parado antes que o processamento seja concluído.
