Benachrichtigungen

Benachrichtigungen informieren Benutzer über CAD- und Bearbeitungsfehler, die im System auftreten. Sie stellen sicher, dass Probleme zeitnah kommuniziert werden, damit entsprechende Maßnahmen ergriffen werden können.

E-Mail-Benachrichtigung

Die E-Mail-Benachrichtigungsfunktion ermöglicht es Benutzern, automatisierte E-Mail-Warnungen für CAD- und Bearbeitungsfehler zu konfigurieren und zu empfangen. Nach der Konfiguration sendet das System automatisch E-Mails an die angegebenen Empfänger, sobald ein relevantes Ereignis eintritt, um sicherzustellen, dass Probleme zeitnah behoben werden.

Um E-Mail-Benachrichtigungen zu konfigurieren, navigieren Sie zu: factory > settings > Notifications

Die Seite Notifications besteht aus den folgenden Abschnitten:

Benachrichtigungseinstellungen

Der Abschnitt Notification settings steuert, wie E-Mail-Warnungen generiert und an Empfänger zugestellt werden.

  • Enable email notifications - Aktiviert das System zum Senden von Benachrichtigungen basierend auf den konfigurierten Bedingungen.

  • Notification recipients - Gibt die E-Mail-Adresse(n) der Empfänger an. Mehrere Empfänger können durch Komma (,) oder Leerzeichen getrennt hinzugefügt werden.

  • Send messages in batch - Fasst mehrere Benachrichtigungen in einer einzigen E-Mail zusammen. Nachrichten, die innerhalb eines definierten Zeitintervalls generiert werden, werden gruppiert, was das E-Mail-Volumen reduziert und Unordnung verhindert.

  • Attach tooling simulation preview where applicable - Hängt eine Bearbeitungssimulationsvorschau an die Benachrichtigungs-E-Mail an, wenn verfügbar. Dies ist nützlich, um bearbeitungsbezogene Probleme direkt aus der E-Mail zu überprüfen.

Das Einbeziehen der Simulationsvorschau kann die E-Mail-Größe erheblich erhöhen.

Mail settings

Der Abschnitt Mail settings definiert die Konfiguration des ausgehenden Mailservers, den das System zur Zustellung von E-Mail-Benachrichtigungen verwendet. Diese Einstellungen müssen korrekt konfiguriert sein, damit Benachrichtigungen erfolgreich gesendet werden können.

  • SMTP server - Gibt die SMTP-Serveradresse an, die zum Senden ausgehender E-Mails verwendet wird (z. B. smtp.gmail.com). Diese Information ist typischerweise im Konto- oder Einstellungsbereich des E-Mail-Clients des Benutzers verfügbar.

  • SMTP port - Definiert die Portnummer, die vom Mailserver zum Senden von E-Mails über das Internet verwendet wird. Übliche Werte sind 465 (SSL) und 587 (TLS).

  • Transport protocol - Gibt das Protokoll an, das zum sicheren Senden von E-Mails von der Anwendung zum Mailserver verwendet wird (z. B. SSL oder TLS).

  • Server needs authentication - Aktiviert die Authentifizierung für den SMTP-Server. Wenn aktiviert, werden Benutzername und Passwort vom Server validiert, bevor E-Mails gesendet werden.

  • User-name - Gibt das E-Mail-Konto an, das zur Authentifizierung beim SMTP-Server und zum Senden von Benachrichtigungen verwendet wird.

  • Password - Gibt das Passwort an, das mit dem zur Authentifizierung verwendeten E-Mail-Konto verknüpft ist.

  • Sender address - Gibt die Absender-E-Mail-Adresse an, die den Empfängern beim Empfang von Benachrichtigungen angezeigt wird.

  • Test…​ - Sendet eine Test-E-Mail zur Überprüfung der Mailserver-Konfiguration.

Klicken Sie auf Test…​, um die Einrichtung zu validieren. Eine Bestätigungsmeldung wird angezeigt, wenn der Test erfolgreich ist; andernfalls wird eine Fehlermeldung angezeigt, die die Ursache des Fehlers angibt.

  • E-Mail-Benachrichtigungen werden im interaktiven Modus nicht ausgelöst (z. B. beim Bearbeiten oder manuellen Speichern eines Teils).

  • Die E-Mail-Zustellung erfolgt im Hintergrund und kann verzögert sein.

  • Benachrichtigungen werden möglicherweise nicht zugestellt, wenn das System oder die Engine gestoppt wird, bevor die Verarbeitung abgeschlossen ist.
