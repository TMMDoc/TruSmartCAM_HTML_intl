Notifications

Notifications informerar användare om CAD- och verktygningsfel som inträffar inom systemet. De säkerställer att problem kommuniceras omgående så att lämplig åtgärd kan vidtas.

Email notification

Funktionen för e-postmeddelanden låter användare konfigurera och ta emot automatiserade e-postaviseringar för CAD- och verktygningsfel. När den är konfigurerad skickar systemet automatiskt e-post till de angivna mottagarna när en relevant händelse inträffar, vilket säkerställer att problem hanteras omgående.

För att konfigurera e-postmeddelanden, navigera till: Fabrik > Inställningar > Meddelanden

Sidan Meddelanden består av följande avsnitt:

Notification settings

Avsnittet Meddelandeinställningar styr hur e-postaviseringar genereras och levereras till mottagare.

  • Aktivera e-postmeddelanden - Aktiverar systemet att skicka meddelanden baserat på de konfigurerade villkoren.

  • Meddelandemottagare - Anger e-post- adress(erna) för mottagarna. Flera mottagare kan läggas till genom att separera dem med ett kommatecken (,) eller ett mellanslag.

  • Skicka meddelande i batch - Kombinerar flera meddelanden till ett enda e-postmeddelande. Meddelanden som genereras inom ett definierat tidsintervall grupperas tillsammans, vilket hjälper till att minska e-postvolymen och förhindrar röran.

  • Bifoga riggningssimulering om tillämpligt + - Bifogar en förhandsvisning av verktygningssimulering till meddelandets e-post när den är tillgänglig. Detta är användbart för att granska verktygningsrelaterade problem direkt från e-postmeddelandet.

Att inkludera simuleringsförhandsvisningen kan öka e-poststorleken avsevärt.

E-postinställningar

Avsnittet E-postinställningar definierar konfigurationen för utgående e-postserver som systemet använder för att leverera e-postmeddelanden. Dessa inställningar måste konfigureras korrekt för att meddelanden ska kunna skickas framgångsrikt.

  • SMTP-server - Anger SMTP-serveradressen som används för att skicka utgående e-post (till exempel smtp.gmail.com). Denna information är typiskt tillgänglig i konto- eller inställningsdelen av användarens e-postklient.

  • SMTP-port - Definierar portnumret som används av e-postservern för att skicka e-post över internet. Vanliga värden inkluderar 465 (SSL) och 587 (TLS).

  • Transportprotokoll - Anger protokollet som används för att säkert skicka e-post från applikationen till e-postservern (till exempel SSL eller TLS).

  • Server kräver autentisering - Aktiverar autentisering för SMTP-servern. När den är aktiverad valideras användarnamnet och lösenordet av servern innan e-post skickas.

  • Användarnamn - Anger e-postkontot som används för att autentisera med SMTP-servern och skicka meddelanden.

  • Lösenord - Anger lösenordet associerat med e-postkontot som används för autentisering.

  • Sender address - Anger avsändarens e-postadress som visas för mottagare när meddelanden tas emot.

  • Test…​ - Skickar ett teste-postmeddelande för att verifiera e-postserverkonfigurationen.

Klicka på Test…​ för att validera inställningen. Ett bekräftelsemeddelande visas om testet lyckas; annars visas ett felmeddelande som indikerar orsaken till misslyckandet.

  • E-postmeddelanden utlöses inte i interaktivt läge (till exempel under redigering eller manuell sparning av en del).

  • E-postleverans hanteras i bakgrunden och kan bli försenad.

  • Meddelanden kanske inte levereras om systemet eller motorn stoppas innan bearbetningen är klar.
